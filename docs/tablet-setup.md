# Tablet / client setup (192.168.1.163 and others)

Every Kodi device uses the **same** config files so they all share one library.

## 1. Find Kodi's userdata folder

| Platform | Path |
|----------|------|
| Android (tablet) | `Android/data/org.xbmc.kodi/files/.kodi/userdata/` |
| Windows | `%APPDATA%\Kodi\userdata\` |
| Linux | `~/.kodi/userdata/` |
| LibreELEC / CoreELEC | `/storage/.kodi/userdata/` |

On Android the easiest way is to enable a file manager add-on or use `adb push`.

## 2. Copy the config files

From `build/kodi-config/` (generated by `./scripts/setup.sh` on the server),
copy into userdata:

- `advancedsettings.xml` — points the library at MariaDB on `192.168.1.177`
- `sources.xml` — adds the SMB sources
- `passwords.xml` — SMB credentials (optional but avoids prompts)

## 3. Restart Kodi and build the library

1. Fully quit and reopen Kodi (so it re-reads `advancedsettings.xml`).
2. Go to **Videos → Files**. You'll see `Movies` and `TV Shows` sources.
3. Set the content type on each source (Movies / TV shows) and choose a scraper.
4. **Update library.** Metadata + watched state now live in the shared DB.

## 4. Add more devices

Repeat steps 1–2 on every device. Because they all point at the same MariaDB
and the same SMB paths, watched status and resume points stay in sync. Only run
a full scan from one device — the rest just read the shared library.

## Troubleshooting

- **Library not syncing** — confirm `advancedsettings.xml` is in userdata and
  Kodi was fully restarted; check the DB host/port are reachable from the device.
- **Can't open shares** — verify `smb://192.168.1.177/Movies` opens and that the
  Samba user/password match `.env`. Kodi defaults to SMBv2/3.
- **Different watched state per device** — a device is likely still on its local
  SQLite library; the `advancedsettings.xml` wasn't picked up.
